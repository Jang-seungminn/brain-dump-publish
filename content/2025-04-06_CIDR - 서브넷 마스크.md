# 📘 CIDR ↔ 서브넷 마스크 ↔ 호스트 수 변환표

|CIDR|서브넷 마스크|사용 가능한 IP 수|브로드캐스트 포함 총 IP 수|
|---|---|---|---|
|/24|255.255.255.0|254|256|
|/25|255.255.255.128|126|128|
|/26|255.255.255.192|62|64|
|/27|255.255.255.224|30|32|
|/28|255.255.255.240|14|16|
|/29|255.255.255.248|6|8|
|/30|255.255.255.252|2|4|
|/31|255.255.255.254|0* (특수 용도)|2|
|/32|255.255.255.255|1 (단일 호스트)|1|

---

## 💡 어떻게 읽으면 되나?

예를 들어 `/26` 이란 표기 나오면 머릿속으로 이렇게 생각하면 된다:

> "아~ 이건 서브넷 마스크 `255.255.255.192`구나.  
> 호스트 주소는 6비트 → 2⁶ - 2 = **62개 사용 가능하겠네!"**

---

## 🔥 추가 팁: 계산 공식 복습

`변화 단위 = 256 - 해당 옥텟 값 전체 IP 수 = 2^(남은 비트 수) 사용 가능 수 = 전체 IP 수 - 2`

예: `255.255.255.192` → 마지막 옥텟 `192 = 11000000` → 남은 비트 = 6  
→ 2^6 = 64개 → 사용 가능 = 62개
