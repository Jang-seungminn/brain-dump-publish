# 🛠️ Chapter 3.1. 윈도우/리눅스에서 IP 설정 실습

---

## ✅ 실습 목표

- 수동 IP 설정하는 법
    
- DHCP로 자동 설정하는 법
    
- 게이트웨이, DNS 직접 넣어보기

---

## ✅ 윈도우에서 수동 IP 설정 방법

### 1. 네트워크 설정 들어가기


`제어판 → 네트워크 및 인터넷 → 네트워크 및 공유 센터 → 어댑터 설정 변경`

또는



`설정 → 네트워크 및 인터넷 → 고급 네트워크 설정`

### 2. 사용하는 어댑터 우클릭 → "속성"

- `Internet Protocol Version 4 (TCP/IPv4)` 클릭 → **속성**

---

### 3. 수동 IP 입력

- **IP 주소**: 예) `192.168.10.100`
    
- **서브넷 마스크**: 자동 입력 또는 `255.255.255.0`
    
- **기본 게이트웨이**: 예) `192.168.10.1`
    
- **DNS 서버**:
    
    - 기본: `8.8.8.8` (Google DNS)
        
    - 보조: `1.1.1.1` (Cloudflare)

> ✅ 확인 → 적용 → 완료!

---

## ✅ 리눅스(Ubuntu 기준)에서 수동 IP 설정

### 1. `nmcli` 명령어 방식 (GUI 없는 환경일 때)


`nmcli con show  # 현재 연결된 네트워크 이름 확인 nmcli con mod "Wired connection 1" ipv4.method manual ipv4.addresses 192.168.10.100/24 ipv4.gateway 192.168.10.1 ipv4.dns "8.8.8.8" nmcli con up "Wired connection 1"`

---

### 2. GUI 환경이면?

- `Settings → Network → Wired → ⚙ → IPv4 → Manual`  
    → 주소, 마스크, 게이트웨이, DNS 입력 후 저장

---

## ✅ 자동 IP 설정 (DHCP)

- 윈도우: IPv4 속성 → **자동으로 IP 주소 받기** 체크
    
- 리눅스:



`nmcli con mod "Wired connection 1" ipv4.method auto nmcli con up "Wired connection 1"`

---

## 🔍 확인 명령어

|OS|명령어|설명|
|---|---|---|
|윈도우|`ipconfig`|현재 IP 정보 확인|
|리눅스|`ip addr`, `ip route`, `nmcli dev show`|IP, 라우팅, DNS 등 확인|
|둘 다|`ping`, `nslookup`, `tracert`|네트워크 상태 테스트|

---

## 🎯 실습 팁

- 실습할 땐 **공유기 DHCP 범위 피해서 수동 설정** 해라  
    → 예: DHCP는 `192.168.0.100~199`,  
    니는 `192.168.0.200` 쓰면 충돌 없음
    
- 수동 설정하고 인터넷 안 되면?
    
    - 서브넷 마스크 틀렸을 수도 있음
        
    - 게이트웨이 빠졌을 수도 있음
        
    - DNS 잘못됐을 수도 있음
