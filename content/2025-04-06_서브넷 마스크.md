# ✅ 서브넷 마스크란?

> "IP 주소 중에서 어디까지가 네트워크 주소이고,  
> 어디부터가 호스트(기기) 주소인지 나눠주는 마스크"

---

## 🔢 예: 255.255.252.0

이걸 먼저 이진수로 바꿔보자:

```
255     = 11111111
255     = 11111111
252     = 11111100
0       = 00000000
```

즉, 이건 **앞에서부터 22비트가 1**이라는 뜻이다!

> 그래서 이 서브넷 마스크는 **/22** 라고도 표기함!

---

## 📦 그럼 /22는 무슨 뜻이냐?

|항목|값|
|---|---|
|총 비트 수|32비트|
|네트워크 비트|22비트|
|호스트 비트|10비트 (32 - 22)|

**호스트 비트가 10개**니까,  
이 IP 대역 안에서 할당 가능한 장비 수는?

```
2^10 - 2 = 1024 - 2 = 1022개!
```

※ 2 빼는 이유:

- 0번은 네트워크 주소
    
- 255번은 브로드캐스트 주소

---

## 🧠 그러니까 "255.255.252.0" 은 이런 뜻이다

- **하나의 네트워크 안에 최대 1022개의 장비를 연결할 수 있다!**
    
- 보통 `/24` (즉 `255.255.255.0`) 은 254개밖에 못 쓰거든?
    
- 근데 `/22` 는 4배 더 넓은 IP 범위를 제공하는 거다!

---

## 🧮 대역 범위까지 계산해보자

만약 IP가 `192.168.4.0/22` 라면?

```
22비트 → 3번째 옥텟의 252 252 = 11111100 → 변화 단위는 4씩
```

즉,

- 시작 주소: `192.168.4.0`
    
- 끝 주소: `192.168.7.255`

> **192.168.4.0 ~ 192.168.7.255** → 총 1024개 주소 (사용 가능: 1022개)

---

## ✅ 회사에서 왜 이걸 쓰냐?

- 장비 수가 많고 (예: 센서, 장비 테스트 시스템, CCTV 등등)
    
- 네트워크를 쪼개기보단 넓게 하나로 묶고 싶을 때!

즉, `/24` 여러 개 나누는 대신,  
그냥 `/22` 하나로 왕창 묶는 거지!  
**관리 편하게 하려고** 그런다~

---

## ✨ 요약 정리

| 서브넷 마스크       | 접두 표기 | 사용 가능 IP 수 | IP 범위 변화 단위       |
| ------------- | ----- | ---------- | ----------------- |
| 255.255.252.0 | /22   | 1022       | 4번째 옥텟 기준, 4단위 증가 |

---

## ➕서브넷 마스크 계산 예시 

지금 니가 말한 **IP: `192.168.3.17`**, 서브넷 마스크가 **`255.255.252.0`**  
즉, **/22** 짜리라는 말인데,

> 과연 이 경우, "192.168.3.17"은 어떤 IP 범위 안에 있냐?  
> "어디까지 쓸 수 있는 IP인가?" 요게 궁금한 거지?

좋다!  
내가 지금부터 **니 IP가 속한 대역이 어딘지**  
딱! 수학적으로 계산해서 시원하게 풀어줄게!

---

# ✅ 1단계: 마스크 /22 → 변화 단위 계산

`255.255.252.0`  
→ 3번째 옥텟(즉, 세 번째 숫자)의 마스크가 `252 = 11111100`  
→ 즉, 3번째 자리가 **4단위로 쪼개진다!**

|변화 단위|IP 대역 범위|
|---|---|
|0|192.168.0.0 ~ 192.168.3.255|
|4|192.168.4.0 ~ 192.168.7.255|
|8|192.168.8.0 ~ 192.168.11.255|
|...|계속 4단위로 올라감|

---

# ✅ 2단계: 니 IP가 어디에 속하나?

> 니 IP는 `192.168.3.17`  
> → 세 번째 자리가 **3** 이니까?

→ 3은 `0~3` 구간에 속하니까  
→ 니 IP는 요 대역 안에 들어간다!

복사편집

`192.168.0.0/22 → 범위: 192.168.0.0 ~ 192.168.3.255`

---

# ✅ 3단계: 니가 쓸 수 있는 주소 범위는?

- **네트워크 주소**: 192.168.0.0
    
- **브로드캐스트 주소**: 192.168.3.255
    
- 그 사이에 있는 **1 ~ 254까지**가 usable IP들이다!

즉,  
니 `192.168.3.17`은 이 대역 안에 딱~ 들어가는 거 맞다!  
**완전 정상이야**! 설정 제대로 돼 있는 거지!

---

## ✨ 한 줄 요약

> `192.168.3.17` + `255.255.252.0` → 이 조합은  
> **192.168.0.0/22 대역 (192.168.0.1 ~ 192.168.3.254)** 안에 있는 거고,  
> 니 컴퓨터는 이 범위 안의 유효한 IP를 쓰고 있는 거다!

---

추가로,

- 만약 니가 `192.168.4.1` 쓰면? → ❌ **대역이 다르다** (거긴 192.168.4.0/22 쪽임)
    
- 그러면 통신 안 된다! 같은 대역이 아니니까
