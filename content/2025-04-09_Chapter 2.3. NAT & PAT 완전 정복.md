# 📚 **Chapter 2.3. NAT & PAT 완전 정복**

---

## ✅ 먼저 질문부터

> 니 집 IP는 `192.168.0.10`인데,  
> 유튜브도 잘 되고, 구글도 되고, 웹사이트 다 되잖아?

근데 생각해봐라… 이 IP는 **사설 IP**라서 **인터넷에선 안 보이는 주소**다.

### ❓ 그럼 인터넷에선 니 컴퓨터를 어떻게 찾는 걸까?

---

## 💡 바로 이걸 해결하는 게 **NAT & PAT**다

---

## ✅ NAT란?

> **NAT = Network Address Translation**  
> → 사설 IP를 **공인 IP로 바꿔주는 기술!**

---

### 예시

- 니 PC IP: `192.168.0.10` (사설)
    
- 공유기 IP: `203.0.113.77` (공인)

> 니가 웹사이트 요청을 날리면?

1. PC → 공유기:  
    "나 구글 접속할래~"
    
2. **공유기(NAT)**: "알겠어~ 니 IP는 공인 IP로 바꿔줄게!"
    
3. 공유기 → 인터넷: "203.0.113.77에서 요청 왔어!"
    
4. 구글 → 공유기로 응답
    
5. 공유기 → 다시 니 PC로 전달!

---

## ✅ PAT란? (포트주소변환)

> PAT = **Port Address Translation**  
> → **하나의 공인 IP로 여러 명이 동시에 인터넷** 쓸 수 있게 해주는 기술!

### 📦 동작 방식

- 내부에 있는 `192.168.0.10`, `192.168.0.11`, `192.168.0.20`  
    → 전부 같은 공인 IP `203.0.113.77`로 나감

> 대신 **포트 번호**로 구분한다!

|내부 주소|내부 포트|외부 포트 (공유기에서 할당)|
|---|---|---|
|192.168.0.10|54321|30001|
|192.168.0.11|54322|30002|
|192.168.0.20|54323|30003|

→ 인터넷에서 돌아오는 응답은 공유기가 포트 보고 다시 올바른 PC로 보내줌!

---

## 🧠 쉽게 비유하면?

> **공인 IP = 건물 주소**  
> **포트 = 각 방 번호**

- 인터넷 택배는 건물 주소로 오고,
    
- 공유기(관리자)가 "아~ 30002호는 철수 방이구나" 하고 가져다줌!

---

## 🛠️ 실무에서 NAT가 하는 일

|역할|설명|
|---|---|
|IP 절약|공인 IP 부족 문제 해결|
|보안 향상|사설 IP는 외부에서 직접 접근 불가|
|가정, 회사 필수|공유기, 방화벽, 라우터 기본 기능|

---

## 📌 NAT 동작 구조 요약

```
[PC (192.168.x.x)] 
    ↓
[NAT 장비 (공유기)]
    ↓ (IP 바꾸기)
[인터넷 (공인 IP)]
```

→ 그리고 돌아올 땐 반대로 다시 번역!

---

## ✨ NAT 종류

|유형|설명|
|---|---|
|Static NAT|1:1 매핑, 항상 고정된 IP로 바꿈|
|Dynamic NAT|풀에서 공인 IP 하나 꺼내 씀|
|PAT (NAT Overload)|하나의 공인 IP + 포트번호로 다 때려박음 (실무 최다!)|

---

## ✅ 정리 요약

|개념|설명|
|---|---|
|NAT|사설 IP → 공인 IP로 바꿔줌|
|PAT|여러 사설 IP → 하나의 공인 IP + 포트로 구분|
|왜 씀?|IP 절약, 보안, 멀티 접속|
|누구 함?|공유기, 방화벽, 라우터|
